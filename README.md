# Secretary-bot + мини-CRM

Бот отвечает за тебя в личке через Telegram Business ("Secretary Mode"),
в твоём стиле, на бесплатном LLM. Плюс мини-CRM внутри Telegram Mini App:
контакты, сегменты-теги, статистика, задачи и напоминания.

## 1. Бесплатный LLM вместо платного API

По умолчанию используется **Groq** — бесплатно, без карты, Llama 3.3 70B,
очень быстрый ответ, провайдер заявляет отсутствие обучения на твоих
данных (важно, раз речь о личной переписке). Лимиты free-тарифа достаточные
для личного бота (десятки запросов в день, не тысячи).

1. Зарегистрируйся на console.groq.com, создай API key.
2. Впиши его в `.env` как `GROQ_API_KEY`.
3. `LLM_PROVIDER=groq` уже стоит по умолчанию — больше ничего делать не нужно.

Если в какой-то момент захочется более "умную" модель — можно переключиться
на `LLM_PROVIDER=gemini` (Google AI Studio, тоже бесплатный тариф, но имей в
виду: Google может использовать твои промпты для обучения моделей, если ты
не в ЕС/UK/EEA — для личной переписки это стоит учитывать). `claude` оставлен
как платная опция, если когда-нибудь понадобится качество ответов выше.

Лимиты бесплатных тарифов у всех провайдеров периодически меняются —
если бот вдруг начнёт получать ошибки 429 (rate limit), глянь актуальные
цифры в документации провайдера.

## 2. Предпосылки по Telegram

- Telegram Premium / Business-аккаунт — без него не появится пункт
  Settings → Telegram Business.
- В @BotFather: `/newbot` → `BOT_TOKEN`; затем Bot Settings → Business Mode → Turn on.
- В своём Telegram: Settings → Telegram Business → Chatbots → username бота →
  выдать права "Reply to messages".
- Узнать свой `OWNER_USER_ID` через @userinfobot.

## 3. Свой стиль

- `style_description.txt` — описание стиля словами.
- `style_examples.txt` — 15-30 твоих реальных сообщений (Telegram Desktop →
  Export chat history → текст без медиа → скопировать свои сообщения).

## 4. Запуск локально

```bash
cd secretary-bot
python -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt
cp .env.example .env   # заполнить значения
python bot.py
```

Один процесс одновременно: long-polling бота (читает business-сообщения),
FastAPI-сервер для мини-аппа на порту из `PORT` (по умолчанию 8000),
и цикл напоминаний.

## 5. Публикация мини-аппа (нужен публичный HTTPS)

Mini App обязательно открывается по HTTPS-адресу — без этого Telegram
не покажет кнопку "Открыть CRM". Два пути:

**Вариант А — бесплатный, без аренды серверов (рекомендуется):**
Гоняешь `bot.py` на своей же постоянно включённой машине (старый ноутбук,
мини-ПК, домашний сервер) и поднимаешь Cloudflare Tunnel — это бесплатно
и даёт публичный HTTPS-адрес без необходимости открывать порты на роутере:

```bash
# один раз
brew install cloudflared   # или скачать бинарник под свою ОС
cloudflared tunnel --url http://localhost:8000
```

Получишь адрес вида `https://что-то.trycloudflare.com` — это и есть твой
`WEBAPP_URL`. Сам бот (polling) не нуждается в туннеле — он сам стучится
в Telegram, входящий HTTPS нужен только для открытия мини-аппа.

**Вариант B — облачный хостинг:**
Render даёт честный бесплатный тариф, но веб-сервис на нём "засыпает" при
простое (первый запрос после паузы — около минуты). Это терпимо для самого
мини-аппа (открыл — подождал — увидел данные), но плохо для постоянно
работающего polling-процесса бота и напоминаний: пока сервис спит, бот не
видит новые сообщения и не шлёт напоминания вовремя. Railway и Fly.io в
2026 уже не "по-настоящему бесплатные" (работают по кредитам/подписке).
Если выбираешь облако — учитывай этот компромисс или будь готов к небольшой
оплате за инстанс, который не засыпает.

После того как адрес есть — впиши его в `WEBAPP_URL` в `.env`, перезапусти
бота, напиши ему `/crm` (от своего OWNER_USER_ID) — придёт кнопка с мини-аппом.

## 6. Что внутри мини-CRM

- **Контакты** — все, кто тебе писал: имя, юзернейм, последнее сообщение,
  счётчик сообщений, теги-сегменты.
- **Задачи** — создаёшь задачу с опциональным сроком; когда срок подходит,
  бот сам присылает тебе напоминание в Telegram (фоновый цикл проверяет
  каждую минуту).
- **Статистика** — число контактов/сообщений/открытых задач, график
  сообщений за 7 дней, распределение по сегментам.

Сегменты (теги) создаются и назначаются контактам через API
(`POST /api/tags`, `POST /api/contacts/{chat_id}/tags`) — в этой версии
интерфейса нет формы создания тега в самом мини-аппе, это первое, что
стоит добавить, если нужна более удобная работа с сегментацией прямо из UI.

## 7. Режимы и безопасность

- `REPLY_MODE=draft` (рекомендуется на старте) — ответы приходят тебе на
  проверку, а не сразу собеседнику.
- `EXCLUDED_CHAT_IDS` — chat_id, которым бот не должен отвечать автоматически
  (близкие, начальник, важные переговоры). chat_id увидишь в логах при первом
  сообщении оттуда.
- Доступ к API мини-аппа проверяется по подписи Telegram `initData` — увидеть
  данные сможет только аккаунт с `OWNER_USER_ID`, даже если кто-то узнает
  адрес мини-аппа.
- `crm.db` (sqlite) хранит всю переписку и контакты локально — учитывай это,
  если сервер/машина общая с кем-то ещё.
